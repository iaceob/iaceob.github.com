Qt 信號 槽

思考一圈下來, 也沒想出能做些什麼, 索性來說說最近學習的 Qt 語言.

在 Qt 中一個重要的概念就是信號與槽機制, 就是這個機制來實現用戶界面與後臺邏輯對接, 一個簡單的實例如下:

信號與槽的連接通過 QObject::connect 來連接, 有四個參數

實現的方式就是某個發射源發射一個信號交給一個接收信號體去接收信號, 在信號實現中做具體實現.
例如上方, 由 ui 中的登入按鈕 (this->ui->btnLogin) 發射點擊信號(事件) (SIGNAL(clicked(bool))) 由當前類接收 (this) 使用 loginSlot 槽做邏輯實現 (SLOT(loginSlot())).

等等.

有沒有感覺很熟悉 ?
是不是在什麼地方見過這種方式 ?
相信你已經想起來了:

其實就和 JQuery 的事件綁定不就是一個意思嘛.

$(‘#btnLogin’) 綁定信號發射源
.bind(‘click’) 綁定事件名 (信號名)
function(){} 匿名函數就相當於 loginSlot() 槽實現

這樣就能很容易理解信號與槽了, 換湯不換藥, 最終還是相同的方式.

漫談設計 數據庫 層級關係

系列文章第一文
數據庫設計中層級關係數據設計

常規型

層級關係會出現在絕大部分系統中, 通常的做法 (也可以說是標準做法) 如下:

id pid name
1 0 home
2 1 linux
3 2 ubuntu
4 1 develop

上述表中所要描述的是如下關係

使用 pid 字段來達到描述層級關係的目的, 但是這種設計方案所帶來的問題:

優點

  • 層級關係維護簡單
  • 擴展性強

缺點

  • 數據不直觀

    如果出現相同名稱 (在設計系統節點時, 不同控制器下可能會出現相同的路由名, 例如: save delete 等), 不容易判斷出當前記錄輸入某個父模塊; 另就算不重名, 也很難根據某一條數據顯示出此條記錄的層級信息.

  • 數據結構構造複雜

    通常需要使用遞歸來實現數據結構分析 (代碼中或者數據庫遞歸).

不要命型

就算死也要死的華麗.
這是一種簡單暴力的方式實現層級關係

一個簡單的數據結構如下, 數據可以美麗成這樣:

id path name
1 0 總部
2 0-1 天朝分部
3 0-1 美利堅分部
4 0-1-2 河北
5 0-1-2 河南
6 0-1-3 華盛頓

美麗到一眼就能看出來想要表達什麼意思.

總部的自標為 0
然後下一級, 天朝分部和美利堅分部, 則繼承總部的 0 為表忠誠, 把上級大旗拿來標榜, 遂變成 0-1
接下來到在下面, 河北 河南 華盛頓 把上級的稱謂拿出來, 為表忠誠, 把上級大旗拿來標榜, 遂變成 0-1-2 與 0-1-3

呵!呵!呵!

好, 現在總部想看看到底有多少手下, 哪些手下.

那麼 就可以這樣
例如在 SqlServer 中就可以使用此種方式進行查詢, 並比較結果

id relpath name
1 0-1 總部
2 0-1-2 天朝分部
3 0-1-2-3 河北
4 0-1-2-4 河南
5 0-1-5 美利堅分部
6 0-1-5-6 華盛頓

也就是把每個人自己的旗幟放到自己的等級上, 然後就能按照這個等級來排個序.

perfect!

好 嚴肅

雖然簡簡單單的就實現了層級關係, 但是這有個非常嚴重的問題!
公司發展的越來越好, 總部一直在想著要換一個更顯地位的地方, 但是原來的地還留著, 處理一些閒雜事, 那這個時候總部搬遷, 新增了個標識是 93204 的總部, 等級仍然是最高 (0) , 這就面臨著所有分部都得將總部的標識給改掉, 然而, 這並不好改.

問:
如何將
0-1 0-1-2 0-1-2 0-1-3
改為
0-93204 0-93204-2 0-93204-3

所以, 這並不是一個很好的方案.

優點

  • 數據直觀
  • 數據結構構造簡單

缺點

  • 不易於調整
  • 擴展性差

中和型

其實就是 不要命型 的翻版, 換湯不換藥

在說這之前, 先看一段數據

這看起來是不是就很像了, 此處的一串數字是地區以及下轄地區的獨立編碼
下轄的編碼從上級的編碼從末尾開始按順序排, 這樣就實現了多層級關係.

然而, 一個大的問題是, 例如北京市轄區 110100 從現有的數據看, 市轄區最做只能容納 100 個區域, 多了就沒法放 (前提是真有 100+ 個區)

在特殊場景的應用效果很好 (例如地區編碼)

優點

  • 數據直觀, 簡單明瞭, 可讀性高
  • 數據結構構造簡單
  • 便於維護， 方便查找以及記憶

缺點

  • 擴展性差
  • 不易於調整

總結

常規型 不要命型 中和型
可擴展 Y N N
易構造 N Y Y
易讀 N Y Y
易改 Y N N
效率 (開發) ↓ - ↑
場景 所有場景 所有場景 (前提是不怕死) 特殊

而每一天， 氣溫都是攝氏三十五度（華氏九十五度）。 據說政府可能有一個政策： 如果氣溫達到三十七度， 在那天其餘的事件， 每個人都可以不工作， 因此他們總是宣佈官方氣溫是三十五度。 我常常騎一個朋友的腳踏車取與慶公園， 在那兒買一杯茶， 然後問工作人員氣溫是幾度。
「三十五度」 他們會說， 並拿報紙搧涼。
「昨天的氣溫是幾度」
「三十五度」
「那你想明天會是幾度」
他們會翻翻白眼， 叫我去喝我的茶。 對他們而言， 那不算一個笑話。 中國也許是世界上唯一由政府控制氣溫的國家， 不過兩年後， 北京氣象站終於開始宣佈真正的氣溫。 當地的報紙為這個發展喝彩， 認為那是邁向向人民說實話的重要一步。

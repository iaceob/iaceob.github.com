模板

模板

模板包装器提供将不同的模板框架统一实现, 目前实现的包装器有

  • beetl
  • enjoy
  • freemarker
  • velocity

模板配置

因不同的模板引擎的配置差异非常大, 分别针对不同的模板引擎有不同的配置, 但是也都有相同的地方, 包括 模板路径 编码 等信息, 因此所有模板引擎的配置均实现自 io.enoa.template.EnoaEngineConfig 剩下的差异化交给不同引擎配置.

一个通用模板引擎配置:

EnoaEngineConfig config = new EnoaEngineConfig() {
  @Override
  public String name() {
    return "main";
  }

  @Override
  public String viewPath() {
    return PathKit.getRootPath().concat("/src/test/beetl");
  }

  @Override
  public boolean debug() {
    return false;
  }

  @Override
  public String datePattern() {
    return null;
  }

  @Override
  public String suffix() {
    return "html";
  }

  @Override
  public Charset charset() {
    return Charset.forName("UTF-8");
  }

  @Override
  public boolean compress() {
    return false;
  }

  @Override
  public EoCompressorFactory compressor() {
    return new HtmlCompressorProvider();
  }
};

  • name()

    该引擎实例名称, 该名称默认为 main, 通常情况一个工程只会存在一个模板引擎, 多个模板引擎实例需要该模板引擎自身的支持, enoa 目前未作处理. 如果需要多个实例可通过创建多个实例实现.

  • viewPath()

    模板根路径

  • debug()

    是否 debug 模式, 需要模板引擎支持

  • datePattern()

    时间格式默认处理方案, 需要模板引擎支持

  • suffix()

    模板引擎加载模板文档默认后缀名

  • charset()

    模板引擎加载模板文档编码

  • compress()

    是否压缩

  • compressor()

    压缩器

配置中需要注意的是 compress() 以及 compressor() 的设置, 该配置实际上并不是模板引擎去实现的, 而是 enoa 提供的方案, 其作用是将模板引擎渲染后的内容进行压缩, 模板提供的压缩器有 HtmlCompressorProvider 以及 YUI Compress 当前仅是测试用法, 在性能等方面还需要进行优化, 不建议生产环境使用.

除以上公共的配置之外, 其他的交由不同的模板引擎的配置处理, 以下是现有的模板引擎配置类

ENGINE CONFIG CLASS
beetl io.enoa.template.provider.beetl.BeetlConfig
enjoy io.enoa.template.provider.enjoy.EnjoyConfig
freemarker io.enoa.template.provider.freemarker.FreemarkerConfig
velocity io.enoa.template.provider.velocity.VelocityConfig

目前压缩器已有的提供者

  • htmlcompressor
  • YUI compressor

这里也列出压缩器的提供者

COMPRESSOR PROVIDER CLASS
htmlcompressor io.enoa.template.compressor.provider.htmlcompressor.HtmlCompressorProvider
YUI compressor io.enoa.template.compressor.provider.yui.YUIProvider

压缩器需要注意, 目前 YUI compressor 尚不能工作.

模板配置会做自动适配, 只要是 EnoaEngineConfig 接口即可, 如果不是相应的模板引擎, 则除 EnoaEngineConfig 之外的配置均使用默认配置.

除了该配置接口外, 可通过设置基本的模板引擎启动配置, 然后使用 EnoaEngine 接口中提供一个 originEngine 的方法, 该方法返回的是该提供者的原始模板引擎类, 若有 enoa 包装器配置中尚未提供的配置, 可通过调用此方法获取原始模板引擎类增加配置.

模板引擎使用

模板引擎同样提供默认管理器 EnoaTemplateMgr, 例如使用启动 enjoy 模板

EnoaTemplateMgr.me()
  .defTemplateFactory(new EnjoyProvider())
  .defConfig(
    new EnjoyConfig.Builder()
    .viewPath(PathKit.getRootPath().concat("/src/test/tpl/enjoy"))
    .build()
);
EnoaEngine engine = EnoaTemplateMgr.me().defEngine()

如果要启动多个或者不同的模板引擎

EnjoyConfig config = new EnjoyConfig.Builder()
  .viewPath(PathKit.getRootPath().concat("/src/test/tpl/enjoy"))
  .build();
EnoaEngine engine = new EnjoyProvider().engine(config);

开发

template

与模板引擎相关接口

  • EoTemplateFactory
  • EnoaEngine
  • EnoaTemplate
  • EoEngineConfig

EoTemplateFactory 为模板引擎提供者, 提供者要求返回 EnoaEngine 引擎, 获取引擎后, 可以从该引擎中获取 EnoaTemplate 最终在该模板中进行渲染. EoEngineConfig 为模板引擎配置信息接口.

调用关系

EoTemplateFactor.engine(EoEngineConfig).template(viewName).render()

compressor

与压缩器相关接口

  • EoCompressorFactory
  • EnoaCompressor

EoCompressorFactory 为压缩器提供者, 提供者要求返回压缩器实现 EnoaCompressor, 通过该压缩器进行压缩.

调用关系

EoCompressorFactory.compressor().compress(html)

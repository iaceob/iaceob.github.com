HTTP 客戶端

HTTP 客戶端

HTTP 客戶端是個尚未完成的組件, 原計劃該組件將會對 HttpClient OkHttp 等 HTTP 客戶端進行封裝, 但是, 這些框架本身都相對完善且使用方式上完全不相同, 若在考慮異步請求, 只會變得更複雜. 因此, 決定 enoa-http 將會對 HttpHelper 進行升級, 暫停 HttpHelper 的升級接入到 enoa 的體系中.

HttpHelper

HttpHelper 是基於 Java 自身的 HttpURLConnection 進行封裝的 HTTP 客戶端, 早期出現的目的是爲了彌補 HttpClient 的高 IO 消耗問題, 解決 Java 爬蟲大量的請求後 HttpClient 在結果分析是高 IO 消耗佔用過多的系統資源, 經歷過每日過百萬級數據的考驗. 並提供簡單的 API.

高消耗也同樣存在 OkHttp 中, 特別是在爬蟲方面, 通常爬蟲程序都會切換代理 IP 解決爬取網站封禁問題, 而 OkHttp 的代理需要在創建 Client 時就需要指定, 後續更改就得重新創建 Client 造成資源的浪費, 太頻繁甚至導致內存溢出錯誤. (當然如果你的代理是固定 IP 就另當別論了)

HttpHelper 地址: http-helper

HeepHelper 將在後續版本中接入.

配置加載器

配置加載器

配置加載器的功效是加載工程中的所有配置信息, 加載器加載配置有三種方式

  • Environment
  • Disk
  • Git

加載方式

Environment

系統環境變量以及工程啓動自定義環境變量, 配置加載實現 io.enoa.conf.impl.EoEnvironmentReader, 變量信息包括

  • System.getenv()
  • System.getProperties()

Environment 加載爲自動掃描, 無需手動設定.

Disk

硬盤加載, 實際上是加載工程 classpath 路徑中以

  • properties
  • yml
  • yaml

結尾的配置文件, 目前尚未開放其他擴展名.

classpath 會自動進行掃描, 如果想要指定一個非 classpath 的配置可以在 load 方法中直接指定路徑即可, 但只能是文件, 且以上方列出的後綴結尾.

Disk 加載爲自動掃描, 無需手動設定加載文件.

Git

Git 方式加載, 是加載遠程 Git 倉庫配置文件, 此加載方式的好處是將配置文件統一管理.

Git 加載需手動設定加載倉庫地址.

目前 Git 倉庫地址僅支持 http/https 協議倉庫地址, 私有倉庫暫未支持.

配置領域

配置領域是針對硬盤文件加載而言的 (Git 加載實際上也屬於硬盤加載).

EnoaConfDomain

private Path path;
private String name;
private Env env;
private Origin origin;
private Kv conf;
  • path 文件路徑
  • name 文件名
  • env 領域 { DEVELOPMENT, PRODUCT, TEST }
  • origin 配置來源 { ENVIRONMENT, DISK, GIT }
  • conf 配置信息

配置領域體現在文件名中, 例如 conf-dev.yml -> conf-dev 那麼該領域就爲 dev.

領域規則約定

DOMAIN VALUE
DEVELOPMENT dev
PRODUCT pro
TEST test

Yml 解析

在配置加載器中, 所有的配置最終都轉換爲鍵值対, Yml 的解析也是如此.

example.yml

info:
  description: testconfig
  url: https://www.example.com
protocols:
   - http
   - https

將會被解析成

{
  "info.description": "testconfig",
  "info.url": "https://www.example.com",
  "protocols": ["http", "https"]
}

Json 解析

Json 因格式開放, 書寫格式有太多不確定因素暫不支持.

使用方式

EnoaConf

使用 EnoaConf 進行配置加載

EnoaConf.load("conf.properties", "conf.json", "conf.yml",
      "https://github.com/mstine/config-repo.git");

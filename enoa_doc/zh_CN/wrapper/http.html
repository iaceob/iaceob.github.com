HTTP 客户端

HTTP 客户端

HTTP 客户端是个尚未完成的组件, 原计划该组件将会对 HttpClient OkHttp 等 HTTP 客户端进行封装, 但是, 这些框架本身都相对完善且使用方式上完全不相同, 若在考虑异步请求, 只会变得更复杂. 因此, 决定 enoa-http 将会对 HttpHelper 进行升级, 暂停 HttpHelper 的升级接入到 enoa 的体系中.

HttpHelper

HttpHelper 是基于 Java 自身的 HttpURLConnection 进行封装的 HTTP 客户端, 早期出现的目的是为了弥补 HttpClient 的高 IO 消耗问题, 解决 Java 爬虫大量的请求后 HttpClient 在结果分析是高 IO 消耗占用过多的系统资源, 经历过每日过百万级数据的考验. 并提供简单的 API.

高消耗也同样存在 OkHttp 中, 特别是在爬虫方面, 通常爬虫进程都会切换代理 IP 解决爬取网站封禁问题, 而 OkHttp 的代理需要在创建 Client 时就需要指定, 后续更改就得重新创建 Client 造成资源的浪费, 太频繁甚至导致内存溢出错误. (当然如果你的代理是固定 IP 就另当别论了)

HttpHelper 地址: http-helper

HeepHelper 将在后续版本中接入.

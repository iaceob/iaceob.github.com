渲染

渲染擴展

這裏說的渲染擴展並非是單個, 此處將列出 Yosart 默認支持的渲染擴展.

在 Yosart 中, 渲染支持列表:

RENDER Yosart support
render YES
renderText YES
renderHtml YES
renderTemplate NO
renderJson NO
renderFile YES
renderXml NO
redirect YES
forward YES
renderError YES

Yosart support 列爲 YES 的渲染爲 Yosart 默認支持的渲染擴展, 其他擴展需要另外的擴展支持.

其中需要特別注意的是 render 以及 forward

  • render

    此渲染並非進行模板渲染, 而是字符串渲染, 可以指定 contentType

  • forward

    渲染擴展需要特別注意的是 forward 渲染, 是不支持插件的, forward 實際上是內部跳轉, 會走到最終跳轉的方法進行渲染.

除 Yosart 提供的渲染接口之外, 也可以添加自定義渲染插件.

自定義很簡單, 只需要實現渲染接口即可, 例如一個簡單的案例:

ExampleRenderExt

public class ExampleRenderExt implements YmRenderExt {

  @Override
  public String renderType() {
    return "example";
  }

  @Override
  public YoRender renderer(Request req, Kv attr, Object... args) {
    return new ExampleRender(req, attr, args);
  }

  @Override
  public String name() {
    return "ExampleRender";
  }

  @Override
  public String version() {
    return "1";
  }
}

ExampleRender

class ExampleRender implements YoRender {

  private final Request req;
  private final String data;
  private final Charset charset;

  TextRender(Request req, Kv attr, Object[] args) {
    this.req = req;
    this.text = (String) args[0];
    this.charset = args[1] == null ? EoConst.CHARSET : (Charset) args[1];
  }

  @Override
  public HttpStatus stat() {
    return HttpStatus.OK;
  }

  @Override
  public String contentType() {
    return "text/plain";
  }

  @Override
  public Header[] headers() {
    return CollectionKit.emptyArray(Header.class);
  }

  @Override
  public ResponseBody render() {
    return ResponseBody.create(this.text, this.charset);
  }

}

在 Yosart 啓動時加入該渲染擴展

Yosart.createServer()
  .ext(new ExampleRenderExt())
  .listen()

Control 以及 Action 中使用:

Renderer.with(req).use("example").render("Some data").end();

模板

模板

模板包裝器提供將不同的模板框架統一實現, 目前實現的包裝器有

  • beetl
  • enjoy
  • freemarker
  • velocity

模板配置

因不同的模板引擎的配置差異非常大, 分別針對不同的模板引擎有不同的配置, 但是也都有相同的地方, 包括 模板路徑 編碼 等信息, 因此所有模板引擎的配置均實現自 io.enoa.template.EnoaEngineConfig 剩下的差異化交給不同引擎配置.

一個通用模板引擎配置:

EnoaEngineConfig config = new EnoaEngineConfig() {
  @Override
  public String name() {
    return "main";
  }

  @Override
  public String viewPath() {
    return PathKit.getRootPath().concat("/src/test/beetl");
  }

  @Override
  public boolean debug() {
    return false;
  }

  @Override
  public String datePattern() {
    return null;
  }

  @Override
  public String suffix() {
    return "html";
  }

  @Override
  public Charset charset() {
    return Charset.forName("UTF-8");
  }

  @Override
  public boolean compress() {
    return false;
  }

  @Override
  public EoCompressorFactory compressor() {
    return new HtmlCompressorProvider();
  }
};

  • name()

    該引擎實例名稱, 該名稱默認爲 main, 通常情況一個工程只會存在一個模板引擎, 多個模板引擎實例需要該模板引擎自身的支持, enoa 目前未作處理. 如果需要多個實例可通過創建多個實例實現.

  • viewPath()

    模板根路徑

  • debug()

    是否 debug 模式, 需要模板引擎支持

  • datePattern()

    時間格式默認處理方案, 需要模板引擎支持

  • suffix()

    模板引擎加載模板文件默認後綴名

  • charset()

    模板引擎加載模板文件編碼

  • compress()

    是否壓縮

  • compressor()

    壓縮器

配置中需要注意的是 compress() 以及 compressor() 的設定, 該配置實際上並不是模板引擎去實現的, 而是 enoa 提供的方案, 其作用是將模板引擎渲染後的內容進行壓縮, 模板提供的壓縮器有 HtmlCompressorProvider 以及 YUI Compress 當前僅是測試用法, 在性能等方面還需要進行優化, 不建議生產環境使用.

除以上公共的配置之外, 其他的交由不同的模板引擎的配置處理, 以下是現有的模板引擎配置類

ENGINE CONFIG CLASS
beetl io.enoa.template.provider.beetl.BeetlConfig
enjoy io.enoa.template.provider.enjoy.EnjoyConfig
freemarker io.enoa.template.provider.freemarker.FreemarkerConfig
velocity io.enoa.template.provider.velocity.VelocityConfig

目前壓縮器已有的提供者

  • htmlcompressor
  • YUI compressor

這裏也列出壓縮器的提供者

COMPRESSOR PROVIDER CLASS
htmlcompressor io.enoa.template.compressor.provider.htmlcompressor.HtmlCompressorProvider
YUI compressor io.enoa.template.compressor.provider.yui.YUIProvider

壓縮器需要注意, 目前 YUI compressor 尚不能工作.

模板配置會做自動適配, 只要是 EnoaEngineConfig 接口即可, 如果不是相應的模板引擎, 則除 EnoaEngineConfig 之外的配置均使用默認配置.

除了該配置接口外, 可通過設置基本的模板引擎啓動配置, 然後使用 EnoaEngine 接口中提供一個 originEngine 的方法, 該方法返回的是該提供者的原始模板引擎類, 若有 enoa 包裝器配置中尚未提供的配置, 可通過調用此方法獲取原始模板引擎類增加配置.

模板引擎使用

模板引擎同樣提供默認管理器 EnoaTemplateMgr, 例如使用啓動 enjoy 模板

EnoaTemplateMgr.me()
  .defTemplateFactory(new EnjoyProvider())
  .defConfig(
    new EnjoyConfig.Builder()
    .viewPath(PathKit.getRootPath().concat("/src/test/tpl/enjoy"))
    .build()
);
EnoaEngine engine = EnoaTemplateMgr.me().defEngine()

如果要啓動多個或者不同的模板引擎

EnjoyConfig config = new EnjoyConfig.Builder()
  .viewPath(PathKit.getRootPath().concat("/src/test/tpl/enjoy"))
  .build();
EnoaEngine engine = new EnjoyProvider().engine(config);

開發

template

與模板引擎相關接口

  • EoTemplateFactory
  • EnoaEngine
  • EnoaTemplate
  • EoEngineConfig

EoTemplateFactory 爲模板引擎提供者, 提供者要求返回 EnoaEngine 引擎, 獲取引擎後, 可以從該引擎中獲取 EnoaTemplate 最終在該模板中進行渲染. EoEngineConfig 爲模板引擎配置信息接口.

調用關係

EoTemplateFactor.engine(EoEngineConfig).template(viewName).render()

compressor

與壓縮器相關接口

  • EoCompressorFactory
  • EnoaCompressor

EoCompressorFactory 爲壓縮器提供者, 提供者要求返回壓縮器實現 EnoaCompressor, 通過該壓縮器進行壓縮.

調用關係

EoCompressorFactory.compressor().compress(html)

Java LINT checkstyle Guide

本文 lint 方案此用 checkstyle

使用方式

这里主要介绍使用 maven 的方式进行配置, 为了最终进行代码审核的目的, 如果是日常开发中为了遵循 lint 的规范来书写代码, 建议透过所使用的 IDE 对应的插件来实现.

IDE 插件

  • Intelij IDEA

    进入设定 File -> Settting -> Editor -> code style -> scheme 导入本文最后所列出的规范, 设定完毕后进行代码便可以根据此规范来格式化.

    此外可在 IDEA 中安装 CheckStyle 插件来实现代码规范检查. File -> Setting -> Plugins 查找 CheckStyle-IDE

  • Eclipse

    安装 Eclipse Checkstyle Plugin 插件

  • Other

    其他 IDE 在对应的插件中心搜寻 CheckStyle.

Maven

Maven 的配置是在项目的 pom.xml 文件中添加 plugin 来实现.

执行 mvn checkstyle:checkstyle 指令便可进行格式校验.

错误级别

在 checkstyle 的文件文件中, 可以通过此配置来设定语法校验的报警级别. checkstyle 允许的错误级别有 error, warning, info

如果配置的是 error 级别的报警, 那么搭配上 failsOnError 为 true 则会打断执行, 错误后会停止. 否则会输出报告 (输出的报告文件在 target/checkstyle-result.xml) 不会停止执行.

Checks

Checker

CheckStyle 配置模块需要在 checker 根模块下. 大部分模块是 TreeWalker 子模块. TreeWalker 通过将每个 Java 源文件分别转换为抽象语法树, 然后将结果交给每个子模块进行操作检查. 例如典型的配置如下:

TreeWalker

CheckStyle 语法校验抽象语法树模组.

AbbreviationAsWordInName

检查目标元素名称中连续大写字母长度, 允许强制执行骆驼命名.

上级: TreeWalker

名称 描述 类型 默认值 版本
allowedAbbreviationLength 允许连续大写字母数量. Integer 3 5.8
allowedAbbreviations 自定义跳过检查缩写, 多个已逗号分隔, 不允许出现空格. Set {} 5.8
ignoreFinal 忽略使用 final 修饰的内容. Boolean true 5.8
ignoreStatic 忽略使用 static 修饰的内容. Boolean true 5.8
ignoreOverriddenMethods 忽略使用 @Override 修饰的方法. Boolean true 5.8
tokens 指定 tokens, 具体的参照官网或者 TokenTypes 类. TokenTypes 5.8

TokenTypes

  • CLASS_DEF
  • INTERFACE_DEF
  • ENUM_DEF
  • ANNOTATION_DEF
  • ANNOTATION_FIELD_DEF
  • PARAMETER_DEF
  • VARIABLE_DEF
  • METHOD_DEF

AbstractClassName

验证抽象类名称.

上级: TreeWalker

名称 描述 类型 默认值 版本
format 自定义格式 Regular Expression “^Abstract.+$” 3.2
ignoreModifier 是否忽略匹配名称的类的抽象 (abstract) 修饰符检查. Boolean False 5.3
ignoreName 没啥用 Boolean False 5.3

AnnotationLocation

检查注解位置.

上级: TreeWalker

名称 描述 类型 默认值 版本
allowSamelineMultipleAnnotations 允许多个注解与目标元素在同一行. Boolean False 6.0
allowSamelineSingleParameterlessAnnotation 允许单个参数或无参数的注解与目标元素位于同一行. Boolean True 6.1
allowSamelineParameterizedAnnotation 允许只有一个参数的注解与目标元素未与同一行. Boolean False 6.4
tokens 指定tokens, 具体的参照官网或者TokenTypes类. TokenTypes 太多不写 6.0

TokenTypes

  • CLASS_DEF
  • INTERFACE_DEF
  • ENUM_DEF
  • METHOD_DEF
  • CTOR_DEF

AvoidStarImport

检查是否使用 * 的导入语句.

上级: TreeWalker

名称 描述 类型 默认值 版本
excludes 排除对某些包检查, 子包不会自动排除. Set {} 3.2
allowClassImports 是否允许*进行导入包. Boolean False 5.3
allowStaticMemberImports 是否允许*进行静态导入包. Boolean False 5.3

CatchParameterName

检查捕获参数 (catch (Exception e) ) 名称是否符合由format属性指定的格式.

上级: TreeWalker

名称 描述 类型 默认值 版本
format 指定格式. Regular Expression “^(e|t|ex|[a-z][a-z][a-zA-Z]+)$” 6.14

ClassTypeParameterName

检查类类型参数的标识符命名规则.

上级: TreeWalker

名称 描述 类型 默认值 版本
format 指定格式标识符. Regular Expression “^[A-Z]$” 5.0

CyclomaticComplexity

检查switch, while等复杂度, 根据指定的检查.

上级: TreeWalker

DeclarationOrder

检查是否根据Java语言的代码约定, 类或接口声明部分按照指定顺序出现.

  1. 类 (静态) 变量. 首先是 public 类变量, 然后是 protected, 然后是 package level (没有修饰符) 最后是 private.
  2. 实例变量. 首先是 public 类的变量, 然后是 protected, 然后是 package level (没有修饰符) 最后是 private.
  3. 构造方法.
  4. 方法.
名称 描述 类型 默认值 版本
ignoreConstructors 是否忽略构造函数. Boolean False 5.2
ignoreModifiers 是否忽略修饰符. Boolean False 5.2

EmptyCatchBlock

检查空的 catch 块.

上级: TreeWalker

名称 描述 类型 默认值 版本
exceptionVariableName 指定异常变量名称, 来跳过检查. String “^$” 6.4
commentFormat 指定 catch 块第一行注释内容来跳过检查. String “.*” 6.4

EmptyLineSeparator

检查标题, 包, 所有导入声明, 字段, 构造函数, 方法, 嵌套类, 静态初始化程序和实例初始化程序之后的空行分隔符.

上级: TreeWalker

名称 描述 类型 默认值 版本
allowNoEmptyLineBetweenFields 字段之间不允许空行. Boolean false 5.8
allowMultipleEmptyLines 允许类成员之间多个空行. Boolean false 6.3
allowMultipleEmptyLinesInsideClassMembers 允许类成员中多个空行. Boolean false 6.18
tokens 指定Tokens 5.8

FileLength

检查类文件行数长度.

上级: Checker

名称 描述 类型 默认值 版本
max 最大行数. Boolean false 3.2
fileExtensions 指定处理文件后缀名. Set All Files 5.0

FileTabCharacter

检查源代码中制表符 (\t) .

上级: Checker

名称 描述 类型 默认值 版本
eachLine 是否报告包含每一行, 或只是第一个实例. Boolean false 5.0
fileExtensions 指定处理文件后缀名. Set All Files 5.0
